# Dropdown

Dropdown presents a list of options that can be used to filter or sort existing content.

## Basic usage

In the most basic case, you can simply put `<bx-dropdown>` and `<bx-dropdown-item>` in a similar manner as putting `<select>`/`<option>` like below, and you'll get Carbon-styled dropdown automatically. You can specify selected items via `value` attribute in `<bx-dropdown>`, too:

```html
<bx-dropdown value="bar">
  <bx-dropdown-item value="foo">Foo</bx-dropdown-item>
  <bx-dropdown-item value="bar">Bar</bx-dropdown-item>
  <bx-dropdown-item value="baz">Baz</bx-dropdown-item>
</bx-dropdown>
```

Often you want to render the dropdown items from your data. Below is an example of [`lit-html`](https://lit-html.polymer-project.org), but similar stories apply to React, Angular, Vue, Backbone, Knockout and many others:

```javascript
html`
  <bx-dropdown>
    ${items.map(
      item => html`
        <bx-dropdown-item value="${item.value}">${item.title}</bx-dropdown-item>
      `
    )}
  </bx-dropdown>
`;
```

## Selection

When user attempts to select a dropdown item, `bx-dropdown-beingselected` event fires on `<bx-dropdown>`, which has the following properties in the event details:

| Property | Description                              |
| -------- | ---------------------------------------- |
| `item`   | The `<bx-dropdown-item>` being selected. |

`bx-dropdown-beingselected` bubbles and is cancelable. Canceling this event means canceling change in selection.

## `<bx-dropdown>` attributes, properties and events

Note: For `boolean` attributes, `true` means simply setting the attribute (e.g. `<bx-dropdown open>`) and `false` means not setting the attribute (e.g. `<bx-dropdown>` without `open` attribute).
